vector

内部是线性存储的可变长数组，当内存不够，开辟新空间会把原有的数据复制过去。

用途
  • 可变长数组
  • 存储图的邻接链
初始化一个临时变量返回

map

内部采用红黑树——一种平衡二叉树实现， 红黑树具有自动排序的功能，因此map内部所有的数据，在任何时候，都是有序的（按key排序）。

用途
  • 需要用到dictionary的时候
  • 如果容量不大，map可能比unordered_map稍好

unordered_map

内部用哈希表实现的map，查找时间复杂度一般是O(1)，但不能保证有序。

与hash_map的比较：（详见：https://blog.csdn.net/u014209688/article/details/95366594）

  1. hash_map 不是标准的，unordered_map是C++11新加入的标准
  2. unordered_map性能优于hash_map
用途：

O(1)的查找时间复杂度(但是hash函数如果不好，最坏可能退化到O(n))

set

内部采用红黑树——一种平衡二叉树实现的集合

特点

  1. set中的元素都是按key排好序的

  2. set集合中没有重复的元素

查找
常用操作：
  • begin() 　　 返回set容器的第一个元素的地址
  • end() 　　　　 返回set容器的最后一个元素地址
  • clear() 　　 删除set容器中的所有的元素
  • empty() 　　　 判断set容器是否为空
  • max_size() 　 返回set容器可能包含的元素最大个数
  • size() 　　　　 返回当前set容器中的元素个数
  • erase(it) 删除迭代器指针it处元素
  • insert(a) 插入某个元素

unordered_set

内部使用哈希表实现，具有O(1)的插入和删除时间复杂度
